## 1) Global polish: typography + spacing rules (stop the UI drifting)

### Typography scale (use everywhere)

Create utility classes so every screen uses the same hierarchy:

**`client/src/index.css`**

```css
@layer utilities {
  .t-h1 { @apply text-[28px] leading-[32px] font-semibold tracking-[-0.02em]; }
  .t-h2 { @apply text-[20px] leading-[24px] font-semibold tracking-[-0.01em]; }
  .t-h3 { @apply text-[16px] leading-[20px] font-semibold; }

  .t-body { @apply text-[15px] leading-[22px]; }
  .t-sub  { @apply text-[13px] leading-[18px] text-muted-foreground; }
  .t-cap  { @apply text-[12px] leading-[16px] uppercase tracking-[0.08em] text-muted-foreground; }
}
```

**Rules**

* Screen title = `.t-h2`
* Card title = `.t-h3`
* Labels = `.t-cap`
* Helper text = `.t-sub`
* Body = `.t-body`

### Spacing system (mobile)

* Screen padding: `px-4` always (16px)
* Card padding: `p-4` (16px) or `p-5` (20px) for primary cards
* Vertical rhythm: `space-y-3` (12px) between cards, `space-y-2` inside cards

## 5) Motion: remove “floaty”, add “confident” (Framer defaults matter)

### Use a single motion spec everywhere

```ts
export const motionSpring = { type: "spring", stiffness: 520, damping: 38, mass: 0.8 };
export const motionFast = { duration: 0.14, ease: [0.2, 0.8, 0.2, 1] };
```

### What to animate (and what NOT to)

✅ Animate:

* section expand/collapse height + opacity (fast)
* bottom sheet in/out (spring)
* progress bar width (fast)
* button press scale (tiny)

❌ Avoid:

* long fades on whole screen
* slow card slides for everything (feels “template”)

Add press feedback to tappable cards:

```tsx
<motion.button whileTap={{ scale: 0.99 }} transition={motionFast} />
```
